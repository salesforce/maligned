  6.  sealed abstract class TAList[F[_, _], A, B] extends Serializable

    A type-aligned cons list.

    A type-aligned cons list.

    In a type-aligned cons list, the "output" (or right) type of a node is equivalent to the "input" (or left) type of the node after it. For example, you could chain functions in a type aligned list like String => Option[Int], Option[Int] => Long, Long => Boolean.

    TANonEmptyList for type-aligned lists that are guaranteed to not be empty.

  12.  abstract class TATuple2K[F[_], G[_]] extends AnyRef

    Equivalent to cats.data.Tuple2K but uses an existential type parameter for the pivot (A) type.

    Equivalent to cats.data.Tuple2K but uses an existential type parameter for the pivot (A) type. This allows instances that share the same F and G types to be considered the same type, even if they don't share the same pivot type. This feature is being used to enable tail-recursive traversal across type-aligned structures such as in TANonEmptyList.reduceLeft and TANonEmptyList.reduceRight.

    Inspired by Tomas Mikula's APair.

  13.  trait TAUnfoldRightNonEmpty[F[_, _], S[_], M[_], L, R] extends AnyRef

    A type-aligned, non-empty right-unfold.

    A type-aligned, non-empty right-unfold. This provides a way to build up a type-aligned sequence by starting with the right-most element (TAUnfoldRightNonEmpty.init) and prepending elements (via TAUnfoldRightNonEmpty.prev).

    F

    The type of elements in the type-aligned sequence (ex: Function1).

    S

    State that can be used to determine the element to prepend. It is type-aligned with the input type of the following element so that you can depend on certain properties of the following element. For example, you could set S equal to org.scalacheck.Gen to ensure that the input type of each element in your list has a Gen instance that you can use.

    M

    an effect type (generally monadic) required to generate each element. For example, you could set M to cats.effect.IO if computing the element to prepend required a database lookup.

    L

    the input type in the overall structure

    R

    the output type of the overall structure
